03 / FEATURED WORK

A hand tracked in pixels, rebuilt in cardboard and servos.

A webcam tracks live hand movement, Python maps the landmarks, and Arduino sends the motion into a servo-driven prototype.

01 / Problem

Close the gap between a gesture and a machine.

I wanted to understand how a camera-based software system could control a physical prototype in real time.

02 / Build

Debug one layer at a time.

The system separates vision, mapping, serial communication, wiring, and mechanics so each failure has somewhere to be found.

03 / What I learned

Physical systems expose vague thinking.

Timing, friction, weak joints, noisy readings, and power constraints turn “working code” into only one part of the answer.

04 / Next version

Make the motion worth trusting.

Lower latency, smoother mechanics, gesture states, calibrated angles, and a sturdier 3D-printed structure.
