Computer-vision robotic hand

A webcam tracks live hand movement, Python maps the landmarks, and Arduino turns the motion into a servo-driven prototype.

CV
Status
Prototype
Focus
Vision to physical motion
Stack
Python, MediaPipe, Arduino

Problem

The goal was to turn a visible gesture into a physical response, making software control something real instead of stopping at the screen.

Build

  • Capture live frames from a webcam.
  • Track hand landmarks.
  • Map movement into servo-friendly values.
  • Send motion values to Arduino.
  • Test the output on a physical prototype.

What I learned

Physical systems expose weak assumptions. Timing, friction, jitter, wiring, and structure all matter once code starts touching hardware.

Next version

  • Lower latency.
  • Smoother motion.
  • Better calibrated angle ranges.
  • Stronger structure.
